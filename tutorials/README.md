# ripopt Tutorial Notebooks

A 15-notebook series that walks from the basics of nonlinear programming
through the specific algorithms and data structures inside ripopt. The
series is self-contained — each notebook builds on the previous one, but
any single notebook is readable on its own.

## Audience

Graduate students and practitioners comfortable with multivariable calculus
and linear algebra who want to understand how a modern primal-dual interior
point NLP solver works end-to-end. No prior experience with ripopt is
assumed.

## Prerequisites

- Python 3.10+ with `numpy`, `scipy`, `matplotlib`, `jupyter`
- `pyomo` and the `pyomo-ripopt` plugin (installed via `pip install
  ./pyomo-ripopt` from the repo root)
- `ripopt` on `$PATH` (via `make install` from the repo root)

## Running

```
cd tutorials
jupyter lab
```

Open any notebook and run all cells. The notebooks are written to execute
top-to-bottom without manual intervention.

## Notebooks

| # | File | Topic |
|---|------|-------|
| 1 | `01_nlp_intro.ipynb` | Introduction to nonlinear programming |
| 2 | `02_unconstrained.ipynb` | Gradient descent and Newton's method |
| 3 | `03_kkt_conditions.ipynb` | KKT conditions and first-order optimality |
| 4 | `04_penalty_augmented_lagrangian.ipynb` | Penalty and augmented Lagrangian methods |
| 5 | `05_sqp.ipynb` | Sequential quadratic programming |
| 6 | `06_barrier_methods.ipynb` | The logarithmic barrier method |
| 7 | `07_primal_dual_ipm.ipynb` | Primal-dual interior point method |
| 8 | `08_kkt_matrix.ipynb` | The KKT (saddle-point) matrix |
| 9 | `09_ldlt_bunch_kaufman.ipynb` | LDLᵀ factorization and Bunch-Kaufman pivoting |
| 10 | `10_inertia_correction.ipynb` | Inertia correction and regularization |
| 11 | `11_filter_line_search.ipynb` | Filter line search |
| 12 | `12_convergence_criteria.ipynb` | Convergence criteria and scaling |
| 13 | `13_restoration.ipynb` | The restoration phase |
| 14 | `14_mehrotra_gondzio.ipynb` | Mehrotra predictor-corrector and Gondzio corrections |
| 15 | `15_ripopt_in_practice.ipynb` | ripopt interface, diagnostics, and architecture |

Notebooks 1–5 are solver-agnostic foundations. Notebooks 6–14 implement
the pieces of a barrier method in Python for pedagogy. Notebook 15 connects
the pieces back to the production ripopt codebase.

## Figures

The PNG files in this directory are generated by cells inside the
notebooks and are committed so the rendered notebooks on GitHub display
correctly without a re-run.

## Re-running as a release smoke test

Every release should re-run all 15 notebooks in place against the new
ripopt build (see the release checklist, Section 6h). Any API drift in
`pyomo-ripopt` or the solver output format will show up as a cell failure
or a visibly wrong plot.
